# Context Engine MCP — V2

> **Goal:** Add intelligence on top of V1 memory. Analyze code, plan tasks, and generate content using personal context + real-world data.  
> **Requires:** V1 fully working (Memory + RAG + Web UI)

---

## 🎯 V2 Goal

Three new modules that use V1 memory as their foundation:

1. **Analysis Module** — understand your codebase with context from your past decisions
2. **Planning/Agents Module** — break goals into tasks, suggest next steps
3. **Generation Module** — create social media posts from your work + world news

**V2 success criteria:**
> Ask "what should I work on next in project X?" and get a grounded answer based on real memory.  
> Generate a LinkedIn post that sounds like you, reflects your actual work, and is relevant to current trends.

## 📦 Module 1 — Analysis Module

### What it does

- Reads local files (codebase) via MCP
- Combines file content + relevant memory entries (past decisions, notes, bugs)
- Surfaces patterns, tech debt, refactoring opportunities
- Returns context Claude can reason over

### MCP Tools

```ts
analyze_file(input: {
  path: string              // absolute path to file or directory
  include_memory?: boolean  // pull related memory entries via RAG (default: true)
}) → {
  content: string           // file/directory content summary
  related_memory: Entry[]   // relevant past decisions, notes, bugs
  suggestions: string       // formatted context for Claude to reason over
}
```

```ts
analyze_project(input: {
  project: string           // project name (resolves path from memory)
  focus?: string            // 'debt' | 'patterns' | 'dependencies' | 'general'
}) → {
  summary: string
  related_entries: Entry[]
  files_analyzed: number
}
```

### How it works

```
1. analyze_project({ project: "wizpress" })
   → look up project in memory → get local path
   → read directory structure + key files
   → RAG search: "wizpress decisions bugs architecture"
   → return: file content + related memory as context string

2. Claude Code receives the context
   → reasons over it
   → surfaces insights, suggestions, debt
```

## 📦 Module 2 — Planning / Agents Module

### What it does

- Takes a goal and breaks it into concrete tasks
- Uses memory to understand current state of a project
- Suggests next actions based on what's pending, blocked, or recently done
- Can create entries in memory automatically (new tasks)

### MCP Tools

```ts
plan_goal(input: {
  goal: string              // "implement auth in project X"
  project?: string          // context scope
  create_tasks?: boolean    // auto-add tasks to memory (default: true)
}) → {
  tasks: Task[]             // broken down steps
  context_used: Entry[]     // memory entries used to inform the plan
}
```

```ts
suggest_next(input: {
  project?: string          // scope to a project, or global
  limit?: number            // default: 5
}) → {
  suggestions: Entry[]      // pending/blocked tasks + reasoning
  blocked: Entry[]
  recently_done: Entry[]
}
```

```ts
run_agent(input: {
  goal: string              // autonomous sub-task
  tools: string[]           // which MCP tools the agent can use
}) → {
  result: string
  actions_taken: string[]
  entries_created: string[]
}
```

### How it works

```
1. suggest_next({ project: "presspulse" })
   → get_entries({ project: "presspulse", status: "pending" })
   → get_entries({ project: "presspulse", status: "blocked" })
   → get_entries({ project: "presspulse", since: "7 days ago", status: "done" })
   → format as prioritized suggestion list

2. plan_goal({ goal: "add dark mode", project: "presspulse" })
   → RAG search: "presspulse UI decisions stack components"
   → Claude breaks goal into steps with that context
   → creates task entries in memory automatically
```

## 📦 Module 3 — Generation Module

### Concept

> The user defines **who they are** (rules + intent) and the system generates posts that combine **their real work** (memory) with **what's happening in the world** (internet/news).

Posts feel authentic because they ARE — they're based on real decisions, real work, real context.

### User Setup (stored in memory)

```ts
set_generation_profile(input: {
  platform: 'linkedin' | 'twitter' | 'newsletter'
  intent: string        // "position myself as an AI engineer building autonomous systems"
  tone: string          // "technical but approachable, no buzzwords, first-person"
  topics: string[]      // ["AI agents", "developer tools", "MCP", "TypeScript"]
  avoid: string[]       // ["politics", "crypto", "AI doom/hype"]
  post_frequency?: string  // "2-3x per week"
  language?: string     // "english" | "portuguese" | "both"
}) → Profile
```

### Generation Flow

```
1. User intent + rules (from profile)
         +
2. Personal context (RAG over memory → recent work, decisions, projects)
         +
3. World context (web search → news, trends on user's topics)
         ↓
4. Claude generates draft posts
         ↓
5. Web UI shows drafts → user reviews, edits, approves
```

### MCP Tools

```ts
fetch_world_context(input: {
  topics: string[]          // pulled from profile if not provided
  since?: string            // 'today' | '3 days' | 'week'
  limit?: number            // articles to fetch (default: 10)
}) → {
  articles: { title, summary, url, published_at }[]
}
```

```ts
generate_posts(input: {
  platform?: string         // defaults to profile platform
  count?: number            // how many drafts (default: 3)
  focus?: string            // "my recent work" | "world news" | "mixed" (default: mixed)
  topic_hint?: string       // optional extra focus e.g. "focus on MCP launch"
}) → {
  posts: {
    content: string
    hook: string            // first line (most important for LinkedIn)
    topics_used: string[]
    memory_sources: Entry[] // which memory entries informed this
    news_sources: { title, url }[]
  }[]
}
```

```ts
save_post(input: {
  content: string
  platform: string
  published?: boolean
  published_at?: string
}) → Entry  // saved to memory as type: 'post'
```

### Web UI — Generation Page

```
/generate
  ├── Profile setup (intent, tone, topics, avoid)
  ├── [Generate Posts] button
  │     → fetches world context
  │     → pulls relevant memory
  │     → generates N drafts
  ├── Draft cards (preview + edit inline)
  └── [Save] / [Mark as Published]
```

## ✅ Decisions Made

| Decision | Choice | Why |
|----------|--------|-----|
| Analysis | Local files + memory context | Full picture: code + history of decisions |
| Generation | Rules + intent + memory + internet | Posts feel authentic, grounded in real work |
| Web search | Brave Search API (or Tavily) | Real-time news without hallucinations |
| Social platform | LinkedIn primary | Professional audience, high value for positioning |
| Post drafts | Multiple drafts per run | User picks best, not locked into one |
| Posts storage | Saved as entries (type: 'post') | Reuses existing memory schema |
