# Context Engine MCP — V1

> **Goal:** Give Claude Code real persistent memory and semantic search via MCP tools.  
> **No OpenRouter** — Claude Code CLI is the LLM. The MCP server only provides context and tools.

---

## 🎯 V1 Goal

A working MCP server with:
- Store and retrieve structured memory
- Semantic search (RAG) over that memory
- Basic Web UI to view and create entries

**V1 success criteria:**
> Ask Claude Code "what did I work on this week?" and it answers with real memory data — no hallucinations.

---

## 🧱 Final Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| MCP Server | Node.js + TypeScript | Official MCP SDK |
| Database | Supabase (PostgreSQL) | Auth + API + hosted |
| Embeddings | pgvector (Supabase) | Native in Postgres, no extra infra |
| Embed Model | `text-embedding-3-small` (OpenAI) | Cheap, fast, great for RAG |
| Web UI | Next.js 15 + Tailwind + Float UI + AOS | SSR + premium design system |
| Package Manager | pnpm | Workspaces for monorepo |

## 🗄️ Database Schema (V1)

### `entries` — central table

```sql
create table entries (
  id          uuid primary key default gen_random_uuid(),
  type        text not null,  -- 'task' | 'note' | 'decision' | 'meet' | 'idea' | 'log'
  title       text not null,
  content     text,
  status      text,           -- 'pending' | 'done' | 'blocked' (for tasks)
  project_id  uuid references projects(id),
  person_id   uuid references people(id),
  tags        text[],
  metadata    jsonb,          -- flexible per entry type
  created_at  timestamptz default now(),
  updated_at  timestamptz default now()
);
```

### `projects`

```sql
create table projects (
  id          uuid primary key default gen_random_uuid(),
  name        text not null,
  company     text,
  stack       text[],
  status      text default 'active',  -- 'active' | 'paused' | 'done'
  description text,
  created_at  timestamptz default now()
);
```

### `people`

```sql
create table people (
  id          uuid primary key default gen_random_uuid(),
  name        text not null,
  role        text,
  company     text,
  email       text,
  notes       text,
  created_at  timestamptz default now()
);
```

### `embeddings` — RAG

```sql
create extension if not exists vector;

create table embeddings (
  id          uuid primary key default gen_random_uuid(),
  entry_id    uuid references entries(id) on delete cascade,
  embedding   vector(1536),  -- text-embedding-3-small dimensions
  content     text,          -- the text that was embedded
  created_at  timestamptz default now()
);

-- Index for fast semantic search
create index on embeddings using ivfflat (embedding vector_cosine_ops)
  with (lists = 100);
```

## 🔧 MCP Tools (V1)

### Memory Tools

```ts
add_entry(input: {
  type: string        // 'task' | 'note' | 'decision' | 'meet' | 'idea' | 'log'
  title: string
  content?: string
  project?: string    // project name (auto-resolved to id)
  tags?: string[]
  status?: string
  metadata?: object
}) → { id, created_at }
```

```ts
get_entries(input: {
  type?: string
  project?: string
  tags?: string[]
  status?: string
  since?: string      // ISO date
  limit?: number      // default: 20
}) → Entry[]
```

```ts
update_entry(input: {
  id: string
  status?: string
  content?: string
  tags?: string[]
}) → Entry
```

### RAG Tools

```ts
search_memory(input: {
  query: string       // natural language question
  type?: string       // filter by entry type
  project?: string    // filter by project
  limit?: number      // default: 5
}) → { entries: Entry[], scores: number[] }
```

```ts
query_context(input: {
  question: string    // "what did I work on this week?"
}) → { answer_context: string }  // formatted context ready for Claude to use
```

### Project & People Tools

```ts
add_project(input: { name, company, stack, description }) → Project
get_projects(input: { status? }) → Project[]

add_person(input: { name, role, company, email? }) → Person
get_people(input: { company? }) → Person[]
```

---

## 🔄 RAG Flow — How It Works

```
1. add_entry(...)
   → saves to entries table
   → auto-embed: calls OpenAI text-embedding-3-small
   → stores vector in embeddings table

2. search_memory({ query: "what did I work on this week?" })
   → embeds the query
   → cosine similarity against embeddings table
   → returns top-5 most relevant entries

3. query_context({ question: "..." })
   → runs search_memory internally
   → formats results as readable context string
   → Claude Code uses that context to answer accurately
```

## ✅ Decisions Made

| Decision | Choice | Why |
|----------|--------|-----|
| LLM | Claude Code CLI (native) | No extra cost, no OpenRouter needed |
| Embed model | OpenAI text-embedding-3-small | Cheap ($0.02/1M tokens), 1536 dims |
| Auto-embed | Yes, on every write | Always indexed, no manual step |
| DB | Supabase + pgvector | Everything in one service |
| Web UI V1 | Read + basic Write | Essential visibility from day one |
| Structure | pnpm monorepo | Share types between MCP server and UI |
